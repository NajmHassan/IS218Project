Docker to Azure Cloud

Docker provides the ability to package and run an application in a loosely isolated environment called a container. To get started you will need to install docker on your machine, create a docker hub account, create an azure account, have Git installed on the machine, and ide text editor such as PyCharm or vscode.

To log in to azure use the docker command $ docker login azure this will take you to your web browser to login to your azure account. After logging in you can close the window. To check what contexts are currently running on the machine use the command $ docker context list which will show you the context you are using for the Docker commands which will have a “*” beside its name showing that is teh active context.

Now to create and ACI context to run containers with.

$ docker context create aci --help

There will be flags shown to pass them and avoid the interactive questions to avoid this and pass the flags use

$ docker context create aci myaci

It will ask which subscription you would like to use if you have one then docker will use that. Now we need to select a resource group. You can choose one that has been created previously or choose “create a new resource group” using the arrow keys. If you choose to create a new one it will create it for you and give the status message that it was created. Now use the $ docker context list command to check contexts and it will show the ACI that was just created.

Running containers on the cloud:

$ docker --context myaci run -d –name web -p 80:80 nginx . Docker will show that is has interacted with ACI by creating a container named “web”.

To switch contexts:

$ docker context use myaci

$ docker ps This command will show the container instances running in the cloud rather than the local machine. You can copy and paste the IP address shown into your browser to show the nginx web server running.

To look at container logs:

$ docker logs web

To stop and remove the container:

$ docker rm web
